What is HTML?

HTML stands for HyperText Markup Language. HTML is the standard markup language used to create web pages. HTML files are text files made up of various HTML elements that consist of various tags enclosed in angle brackets like <html> and <table>. Web browsers interpret these elements as different objects like lists, links, and pictures and will display them as such. Try copying the code to the right into a new text file. Save this file with a .html extension and open it in a web browser.

Sample HTML Code

<!DOCTYPE html>
<html>
 <body>
  <h1>HTML Example Heading 1</h1>
  <p>HTML Example Paragraph.</p>
  <h2>HTML Example Heading 2</h2>
  <ul>
   <li>List Item 1</li>
   <li>List Item 2</li>
  </ul>
 </body>
</html>

HTML Elements

HTML elements are the building blocks of web pages. Each HTML element has different possible attribute fields, some are required and some are optional. It is left to the developer to determine which optional tags they require. As shown in the image to the right, each attribute name requires a corresponding attribute value, typically this value needs to be represented by a String and therefore require surrounding quotation marks. Below is a list of commonly used HTML elements and their descriptions.

Common Elements

Tag Description
<html> Declares the file as written in HTML
<head> Defines the page's head, used for page metadata
<title> Defines the page's title
<body> Defines the page's body, containing the pages content
<h1> Defines a header element (h2-h6 define smaller headings)
<ul> Defines a unordered (bulleted) list element
<ol> Defines a ordered (numbered) list element
<div> Defines a divider element, used a a container for content
<p> Defines a paragraph element
<table> Defines a table element
<th> Defines a table heading element
<tr> Defines a table row element
<img> Defines a image element (requires "src" attribute)
<a> Defines a anchor (link) element (requires "href" attribute)

Block vs. Inline Elements

BLOCK

A block element is an element that takes up the full width available, and has a line break before and after it.

INLINE

An inline element only takes up as much width as necessary, and does not force line breaks.

Breaks can be added by inserting a <br \> element.


HTML Tree Representation



An HTML document can be easily represented as a tree data structure. In this tree, each HTML element acts as a node and nesting one element inside another creates a parent-child relationship between those elements. This allows for content to be more easily divided up. The <html> element will always be the root of this tree and it will have 2 children, <head> and <body> This representation of an HTML document is great for things like styling with CSS and user interaction with JavaScript


HTML Entities

Some characters in HTML are reserved for markup syntax. For example, let's say you wanted to use the less than (<) or greater than (>) inside a of mathematical expression you are displaying. Those symbols could be misinterpreted as defining the tag of an element, and end up ruining the page display. To avoid this, HTML defines codes for these escape characters. These codes follow the format below.

&entity_name; OR &#entity_number;
